# Резервное копирование

Этот раздел описывает план настройки резервного копирования.

Основной лист Excel:

`it-templates_2026_04_06.xlsx` → `Бэкапы`

## Текущее состояние

В Excel создан список объектов, которые нужно включить в резервное копирование.

Фактическая схема бэкапов пока требует настройки.

## Что нужно резервировать

В план резервного копирования входят:

- ВМ Windows Server;
- Active Directory;
- файловый сервер;
- сайт `mkdk-studsovet.ru`;
- конфигурации сетевого оборудования.

## План работ по настройке бэкапов

### Этап 1. Определить хранилище

Нужно выбрать, где будут храниться резервные копии:

- внешний диск;
- NAS;
- отдельный сервер;
- облачное хранилище;
- комбинированная схема.


Нужно определить частоту копирования:

- ежедневные копии для критичных данных;
- еженедельные копии для полной системы;
- копии после важных изменений;
- отдельные копии конфигураций сетевого оборудования.

### Этап 3. Настроить копирование серверов

В первую очередь нужно настроить резервное копирование:

- виртуальной машины `ME-FS-DOM`;
- Active Directory;
- файлового сервера;
- базы данных, если она используется.

### Этап 4. Настроить копирование сайта

Для сайта нужно определить:

- где лежит исходный код;
- где лежит опубликованная версия;
- как делать копию файлов;
- нужно ли копировать базу данных;
- где хранить резервную копию.

### Этап 5. Настроить копирование конфигураций сети

После каждого изменения сетевого оборудования нужно сохранять конфигурацию.

### Этап 6. Проверить восстановление

Бэкап считается рабочим только после проверки восстановления.

Проверку нужно фиксировать в Excel.

## Что нужно заполнить в Excel

На листе `Бэкапы` нужно заполнить:

- тип копии;
- частоту;
- хранилище;
- ответственного;
- дату последней копии;
- дату последней проверки восстановления;
- статус;
- комментарий.

## Правило безопасности

Пароли и секретные ключи нельзя хранить вместе с резервными копиями.
