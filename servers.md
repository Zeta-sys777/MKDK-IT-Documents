# Серверы

Этот раздел описывает серверную инфраструктуру колледжа.

Основной лист Excel:

`it-templates_2026_04_06.xlsx` → `Серверы`

## Текущие серверы

### MKDK

- Тип: Физический сервер.
- Инвентарный номер: Отсутствует.
- ОС или гипервизор: Главная Proxmox (гостевая Windows Server для AD).
- CPU: CPU - Intel Xeon E5 2.40Ghz.
- RAM: 79 GB ( 2 32 GB плашки, 2 8 GB плашки).
- Диски: 2 HDD (1 диск на 500гб Hitachi - серийный номер: JP1572FL071UYK , 500 гб STAS серийный номер: Z2AERZNT).
- IP-адрес: `192.168.0.185`.
- Роли: Контроллер виртуальных машин.
- Расположение: 315 кабинет, 3 этаж.
- Статус: В работе.
- Комментарий: Корпус открыт, могут что-то украсть, нужно следить внимательно.

### ME-FS-DOM

- Тип: Виртуальная машина.
- Инвентарный номер: Отсутствует.
- ОС или гипервизор: Windows Server 2016.
- CPU: CPU - Intel Xeon E5 2.40Ghz.
- RAM: 10 GB.
- IP-адрес: `192.168.0.194`.
- Роли: Контроллер Домена.
- Расположение: Proxmox.
- Статус: В работе.
- Комментарий: Развернут внутри физического сервера как выделенная среда.


## Proxmox

Физический сервер `MKDK` используется как платформа Proxmox.

Через Proxmox развёрнута виртуальная машина `ME-FS-DOM`.

## Windows Server

Виртуальная машина `ME-FS-DOM` работает на Windows Server 2016 и используется как контроллер домена.

## Риски

Текущий риск по физическому серверу:

- корпус сервера открыт;
- оборудование требует контроля;
- нужно исключить несанкционированный физический доступ;
- желательно закрыть корпус и ограничить доступ к серверной зоне.

## Что нужно сделать

Планируемые действия:

1. Уточнить точную модель CPU.
2. Проверить состояние дисков.
3. Описать схему хранения данных.
4. Настроить резервное копирование Proxmox и Windows Server.
5. Проверить восстановление виртуальной машины из бэкапа.
6. Описать порядок доступа к Proxmox.
7. Не хранить пароли в документации.
