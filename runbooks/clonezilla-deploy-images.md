# Инструкция: развёртывание образов через Clonezilla

Этот runbook описывает способ удалённого клонирования эталонного образа на другие ПК колледжа с помощью Clonezilla.

Инструкция нужна для массовой подготовки учебных компьютеров после настройки эталонного ПК.

## Назначение

Clonezilla используется для того, чтобы:

- создать образ эталонного ПК;
- сохранить образ на сервере или сетевом хранилище;
- быстро развернуть одинаковую систему на другие учебные ПК;
- ускорить подготовку кабинетов;
- снизить количество ручной настройки после переустановки.

## Где фиксировать работы

После создания или развёртывания образа нужно обновить:

- `it-templates_2026_04_06.xlsx` → `Журнал работ`;
- `it-templates_2026_04_06.xlsx` → `Оборудование`;
- `inventory.md`, если изменилось состояние оборудования;
- `work-log.md`, если нужно отразить важное изменение в документации.

## Рекомендуемая схема

Для колледжа подходит схема:

1. На одном ПК готовится эталонная система.
2. С эталонного ПК создаётся образ через Clonezilla.
3. Образ сохраняется на сервере или внешнем сетевом хранилище.
4. Другие ПК загружаются с Clonezilla.
5. Образ разворачивается на выбранные компьютеры.
6. После развёртывания каждый ПК получает уникальное имя.
7. ПК подключается к домену `MKDK`.
8. Результат фиксируется в журнале работ.

## Требования

Перед использованием Clonezilla нужно подготовить:

- эталонный ПК;
- загрузочную флешку Clonezilla или PXE-загрузку;
- сетевое подключение;
- место для хранения образов;
- список ПК, на которые будет разворачиваться образ;
- доступ к домену `MKDK`;
- ответственного исполнителя.

## Подготовка эталонного ПК

На эталонном ПК нужно настроить:

- Windows;
- драйверы;
- базовые учебные программы;
- сетевые настройки;
- доступ к нужным ресурсам;
- локальные настройки системы;
- обновления;
- очистку временных файлов.

Перед снятием образа нужно проверить:

- система загружается без ошибок;
- нет лишних пользовательских данных;
- нет сохранённых паролей;
- нет временных файлов;
- ПК корректно работает в сети;
- установлен нужный набор программ.

## Важное правило перед снятием образа

Эталонный образ не должен содержать:

- личные файлы;
- пароли;
- токены;
- сохранённые сессии;
- приватные ключи;
- лишние учётные записи;
- данные студентов или сотрудников.

## Создание образа

Общий порядок:

1. Подключить эталонный ПК к сети.
2. Загрузить ПК с Clonezilla.
3. Выбрать режим работы с образом.
4. Выбрать место сохранения образа.
5. Указать понятное имя образа.
6. Запустить создание образа.
7. Дождаться завершения.
8. Проверить, что образ сохранился.
9. Зафиксировать создание образа в журнале работ.

## Рекомендуемое имя образа

Имя образа должно быть понятным.

Пример:

```text
MKDK-WIN-STUD-BASE-2026-04-06
```

Формат:

```text
MKDK-<ОС>-<НАЗНАЧЕНИЕ>-<ДАТА>
```

Примеры:

```text
MKDK-WIN-STUD-BASE-2026-04-06
MKDK-WIN-TEACHER-BASE-2026-04-06
MKDK-WIN-LAB315-2026-04-06
```

## Развёртывание образа на другие ПК

Общий порядок:

1. Проверить, что целевой ПК есть в инвентаризации.
2. Убедиться, что с ПК можно удалить текущие данные.
3. Загрузить ПК с Clonezilla.
4. Выбрать восстановление из образа.
5. Выбрать нужный образ.
6. Выбрать целевой диск.
7. Подтвердить развёртывание.
8. Дождаться завершения.
9. Перезагрузить ПК.
10. Проверить загрузку Windows.
11. Переименовать ПК по формату из инвентаризации.
12. Подключить ПК к домену `MKDK`.
13. Проверить вход доменного пользователя.
14. Обновить Excel и журнал работ.

## Формат имени ПК после развёртывания

Используется формат из инвентаризации:

```text
STUD<номер>-<кабинет>
```

Примеры:

```text
STUD1-315
STUD2-315
STUD10-315
TEACHER315
```

## Подключение к домену после клонирования

После развёртывания образа ПК нужно подключить к домену.

Для этого использовать инструкцию:

`runbooks/join-pc-to-domain.md`

Важно: каждый ПК должен иметь уникальное имя. Нельзя подключать к домену несколько компьютеров с одинаковым именем.

## Что проверить после развёртывания

После клонирования нужно проверить:

- загрузку системы;
- имя компьютера;
- сетевое подключение;
- получение IP-адреса;
- доступность DNS;
- подключение к домену;
- вход доменного пользователя;
- работу учебных программ;
- состояние устройства в инвентаризации.

## Что записывать в журнал работ

В журнале нужно указать:

- дату и время;
- исполнителя;
- категорию `ПК`;
- объект, например `Кабинет 315`;
- какой образ использовался;
- на какие ПК он был развёрнут;
- итог;
- следующее действие;
- комментарий.

Пример описания работы:

```text
Развёрнут образ MKDK-WIN-STUD-BASE-2026-04-06 на ПК STUD1-315, STUD2-315, STUD3-315.
```

## Риски

Основные риски:

- можно случайно стереть данные на неправильном диске;
- можно развернуть устаревший образ;
- можно получить одинаковые имена ПК;
- можно забыть подключить ПК к домену;
- можно не обновить инвентаризацию;
- можно потерять важные данные, если не проверить целевой ПК заранее.

## Правила безопасности

Перед развёртыванием образа нужно убедиться, что на целевом ПК нет важных данных.

Нельзя хранить образы с личными файлами, паролями и приватными данными.

Доступ к образам должен быть только у IT-департамента.

## Когда обновлять образ

Образ нужно обновлять, если:

- изменился набор программ;
- изменились драйверы;
- обновилась базовая конфигурация;
- изменились требования к учебному кабинету;
- старый образ начал создавать проблемы при развёртывании.

## Минимальный чек-лист

Перед клонированием:

- эталонный ПК проверен;
- образ подписан понятным именем;
- целевые ПК определены;
- данные на целевых ПК можно удалить;
- сеть работает;
- ответственный назначен.

После клонирования:

- ПК загружается;
- имя ПК уникальное;
- ПК подключён к домену;
- вход пользователя проверен;
- инвентаризация обновлена;
- журнал работ заполнен.
